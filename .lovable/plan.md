

## الوضع الحالي
أنت فاتح FileZilla ومتصل بـ VPS، وتريد:
1. حذف ملفات النظام القديم من السيرفر
2. رفع ملفات النظام الجديد مكانها

## ⚠️ تحذير مهم قبل الحذف

**لا تحذف هذه الملفات أبداً** (لو موجودة):
- `.env` — يحتوي كلمات السر وروابط القاعدة
- `uploads/` — ملفات المستخدمين (صور، تسجيلات)
- `backups/` — النسخ الاحتياطية للقاعدة
- `node_modules/` — لو حذفته لازم تعمل `npm ci` ثاني عبر SSH

## الخطوات في FileZilla

### 1️⃣ اعرف وين النظام القديم
في الجانب الأيمن (Remote)، تنقّل بين هذه المسارات الشائعة:
- `/var/www/html/`
- `/var/www/hulul/` (لو سبق ركّبت بسكربتنا)
- `/home/USERNAME/public_html/`
- `/root/` أو مجلد المشروع القديم

### 2️⃣ احذف الـ frontend القديم فقط
ادخل إلى مجلد الـ frontend (مثلاً `/var/www/hulul/public/` أو `/var/www/html/`):
- اضغط Ctrl+A لاختيار كل الملفات
- Right-click → **Delete**
- أكّد الحذف

> **ملاحظة**: لو شفت ملف `.htaccess` احذفه معهم، النظام الجديد ما يحتاجه.

### 3️⃣ ابنِ النظام الجديد على جهازك
افتح Terminal/CMD في مجلد المشروع على جهازك:
```bash
npm ci
npm run build
```
سيتولّد مجلد `dist/` فيه الملفات النهائية.

### 4️⃣ ارفع `dist/` عبر FileZilla
- في الجانب الأيسر (Local): افتح مجلد `dist/`
- في الجانب الأيمن (Remote): ادخل المجلد الفارغ (مثلاً `/var/www/hulul/public/`)
- اختر **محتوى** `dist/` (Ctrl+A) — مش المجلد نفسه
- اسحبه للجانب الأيمن

### 5️⃣ للـ backend (يحتاج SSH — مو FileZilla)
الـ backend مش ممكن تشغيله بـ FileZilla لوحده. لازم بعد الرفع:
```bash
ssh root@VPS_IP
cd /var/www/hulul/backend
npm ci --omit=dev
pm2 restart all
```

## أحتاج منك توضيح قبل ما أبدأ

عشان أعطيك خطوات دقيقة، لازم أعرف:

1. **شو موجود فعلياً على السيرفر الآن؟** (نظام قديم WordPress؟ موقع HTML بسيط؟ نسخة قديمة من نفس التطبيق؟)
2. **شو مسار المجلد اللي فيه الملفات القديمة في FileZilla؟** (انسخ المسار من الجانب الأيمن)
3. **هل ركّبت سكربت `01-server-setup.sh` على السيرفر؟** (لو لأ، نحتاج SSH أولاً قبل أي رفع)
